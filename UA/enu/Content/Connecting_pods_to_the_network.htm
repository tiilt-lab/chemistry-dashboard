Connecting pods to the network using Terminal

Your pods are preconfigured to work with Discussion Capture. However, you can use the following instructions if you need to configure the pods to use a different network connection to access the Discussion Capture server.

If you need to add or remove pods from the Discussion Capture Dashboard, please see "Adding or disconnecting pods in the Dashboard."

  1. Open a Terminal session on the pod.

    Using ssh

    If a pod is already connected to a network and you know its IP address, you can connect using ssh.

    1. Open a Terminal window on macOS or Linux.

    2. Type ssh respeaker@xxx.xxx.xx.xx, where xxx.xxx.xx.xx is the IP address of the pod.

    Using a serial connection over USB

    1. Using the supplied USB cable, connect the microUSB port at the top of the pod to a USB port on your macOS or Linux computer.

    2. Open a Terminal window on macOS or Linux.

    3. Type the following command to display the path to the connected pod:

      macOS: ls /dev/cu.usb*

      Linux: ls /dev/ttyACM*

    4. Type the following command to establish a terminal session with the pod:

      macOS: screen /dev/cu.usbModem1141 115200

      Linux: screen /dev/ttyACM0 115200

    5. Sign in to the pod using the device's username and password.

  2. Choose wired Ethernet or Wi- Fi connections:

    Wired Ethernet

    1. In Terminal, type sudo nmtui.

    2. Use the arrow keys to select Edit a Connection and then press Enter.

    3. Use the arrow keys to highlight your Wi-Fi network and <Delete>.

    4. Press Enter to delete the selected network, and then select Delete in the confirmation dialog.

    5. Repeat steps C and D above to remove all Wi-Fi networks.

    6. Connect the pod to your network with an Ethernet cable.

    7. Press Esc twice to exit nmtui.

    Wi-Fi

    1. In Terminal, type sudo nmtui.

    2. Use the arrow keys to select your Wi-Fi network, and then press Enter.

    3. Type your network key, and then press Enter.

    4. Press Esc twice to exit nmtui.
